# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in **CybrScrape**, please report it privately so we can address it before it is publicly disclosed.

**Do not open a public GitHub issue for security vulnerabilities.**

### How to report

Email **platform@cybrflux.online** with:

- A clear description of the issue
- Steps to reproduce, or a proof of concept
- The version affected
- Your contact info for follow-up

You will receive an acknowledgment within **3 business days**. We aim to provide an initial assessment within **7 days** and a fix or mitigation within **30 days** for confirmed vulnerabilities, depending on severity.

## Supported versions

Only the latest minor release line receives security updates. Older releases are best-effort.

## Disclosure

We follow coordinated disclosure: we'll work with you on a patch, credit you in the release notes (unless you'd rather stay anonymous), and publish an advisory after the fix ships.

## Scope

In scope: code in this repository.
Out of scope: third-party dependencies (report upstream), social engineering, denial-of-service via excessive requests.
